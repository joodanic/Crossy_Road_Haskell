mapaValido :: Mapa -> Bool #

Esta é a função principal. É uma conjunção das funções : contiguosR , converteMT, terrenosImp, verificaNenhumAll , verificaComp , converteMLO , verificaCarroAll , verificaRios , pegaRios , verificaER , pegaEst , pegaRel, onde o mapa só é válido se todas estas funções forem válidas.

converteMT :: Mapa -> [Terreno] #

Função que retorna uma lista dos terrenos do mapa. Esta função permite uma utilização mais fácil dos terrenos do mapa, como por exemplo nas funções contiguosR e pegaRios.

converteMLO :: Mapa -> [[Obstaculo]] #

Função que retorna uma lista das listas de obstáculos do mapa. Esta função, assim como a de cima, também permite uma mais fácil utilização das listas de obstáculos, como por exemplo na função verificaComp.

contiguosR :: [Terreno] -> Bool #

Esta função é a responsável pela verificação de rios contíguos. Esta função recebe a lista de terrenos de um mapa(a partir da função converteMT) e verifica se o primeiro terreno é um rio e a cabeça da cauda da lista é um rio. Para além disso, também verifica se esses dois rios têm direções diferentes com o auxílio da função verificaRC.

checkVelocidade :: Terreno -> Velocidade #

Esta função retira a velocidade de um Rio ou de uma Estrada para auxiliar na função que verifica se dois rios contíguos têm direções diferentes.

verificaRC :: Terreno -> Terreno -> Bool #

Esta função verifica se dois Rios contíguos têm direções diferentes, com o auxílio da função checkVelocidade.

terrenosImp :: [(Terreno, [Obstaculo])] -> Bool #

Esta função tem é composta por 3 funções, a verificaA, que verifica se as Arvores estão na Relva, a verificaC que verifica se os Carros estão na Estrada e a verificaT que verifica se os Troncos estão no Rio.

Esta função recebe um terreno e uma lista de obstáculos e verifica se as Arvores se encontram na Relva ou não.

Esta função recebe um terreno e uma lista de obstáculos e verifica se os Troncos se encontram no Rio ou não.

verificaC :: Terreno -> [Obstaculo] -> Bool #

Esta função recebe um terreno e uma lista de obstáculos e verifica se os Carros se encontram na Estrada ou não.

verificaNenhumAll :: [(Terreno, [Obstaculo])] -> Bool #

Esta função recebe a lista de Terrenos e lista de Obstáculos e confere se existe um Obstaculo Nenhum no mapa. Esta função chama a verificaNenhum que verifica se existe um Obstaculo Nenhum na primeira lista de Obstáculos. Se não existir na 1ª lista, esta função verifica se existe no resto das listas de Obstáculos, chamando novamente a verificaNenhum. Se no final das listas de Obstáculos, não houver, isso invalida o mapa ser válido.

verificaNenhum :: [Obstaculo] -> Bool #

Esta função verifica se existe pelo menos um Obstaculo Nenhum numa dada lista de Obstáculos. Com a auxiliar somaN que dá o número de Nenhum numa lista de Obstáculos, basta verificar se esse número é maior ou igual a 1.

somaN :: [Obstaculo] -> Int #

Esta função auxilia a verificaNenhum de tal modo que, isto dá o número de Obstáculos Nenhum numa lista de Obstáculos. Com isto, ela retorna o número de Nenhum numa lista de Obstáculos.

verificaComp :: Int -> [[Obstaculo]] -> Bool #

Esta função é responsável por verificar se a largura do mapa corresponde ao comprimento do mapa, onde este é dado pelo tamanho da lista de Obstáculos. Com o auxílio da função soma, esta função calcula o comprimento de todas as listas de obstáculos do mapa, verificando que esse valor é igual à largura do mesmo.

Esta função verifica se, numa lista de Obstáculos, existe um Tronco de tamanho 5, com auxílio das funções: verificaTronco e pegaTronco.

verificaTronco :: [[Obstaculo]] -> Bool #

Esta função auxilia a verificaTroncoAll de tal modo que, esta compara se o resultante da tamanho da pegaTronco é >5. Se for esse o caso, então existe um Tronco de comprimento 5 ou mais, o que invalida o mapa. Caso contrário, o mapa é válido.

verificaRios :: [[Terreno]] -> Bool #

Esta função verifica se, dados uma lista de listas de terrenos, se existe uma lista tal que tenha 4 rios contíguos. Isso é possível ao comparar o tamanho de cada lista resultante do agrupamento de rios pela função pegaRios a 4. Se esse valor for maior que 4, então o mapa torna-se inválido, caso contrário, o mapa é válido.

pegaRios :: [Terreno] -> [[Terreno]] #

Esta função é a responsável por, dada uma lista de Terrenos, agrupar numa lista de listas os Rios contíguos. Esse resultado é possível ao utilizar as funções takeWhile e dropWhile, ou seja, ao percorrer a lista, a takeWhile vai criando uma lista de Rios enquanto eles forem seguidos. Quando houver um Terreno que não seja Rio, a dropWhile vai atravessando a lista até encontrar novamente um Rio, onde a takeWhile começa a criar uma lista de Rio, juntando à outra lista de Rio, já existente.
